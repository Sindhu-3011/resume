1
Install Python
The language the app is written in
a

Open your browser and go to python.org/downloads — it will automatically show the latest version for Windows. Click the big yellow "Download Python 3.x.x" button.

b

Run the downloaded file. On the first screen, tick the box that says "Add Python to PATH" at the bottom. This is critical — do not skip it.

🔴 Tick "Add Python to PATH" before clicking Install Now. If you miss this, the app will not work and you will need to reinstall Python.
c

Click Install Now and wait for it to finish. Click Close when done.

✅
Verify it worked Press Windows key + R, type cmd, press Enter. In the black window, type the command below and press Enter. You should see something like Python 3.11.x.
Command Prompt
C:\> python --version
2
Install PostgreSQL
The database that stores all resume data
a

Go to postgresql.org/download/windows and click "Download the installer". Download the latest version for Windows x86-64.

b

Run the installer. Click Next through all screens until it asks for a Password. Set the password to exactly: postgres

⚠️ Write this password down. The app's default setup expects the password postgres. If you choose a different password, you will need to update the .env file in Step 6.
c

Keep the port as 5432 (default). Click Next through the rest and finish the installation. Skip the Stack Builder at the end — click Finish without ticking the checkbox.

✅
Verify it worked Open the Windows Start menu and search for pgAdmin 4. If it opens without errors, PostgreSQL is installed correctly.
3
Install Ollama
Runs the AI model locally on your PC
a

Go to ollama.com and click Download for Windows. Run the downloaded installer and follow the prompts. Ollama runs quietly in the background — you will see its icon in the system tray (bottom-right of your screen).

✅
Verify it worked Open Command Prompt and run this command. You should see a short response.
Command Prompt
C:\> ollama --version
4
Set up the app
Install the app's Python packages
a

Copy the app folder to a simple location, e.g. C:\ResumeApp. Avoid paths with spaces or special characters.

b

Open Command Prompt. Navigate to the app folder by typing the command below (replace the path if your folder is in a different location) and press Enter.

Command Prompt
C:\> cd C:\ResumeApp\resume_profile_saved_list
c

Now install all required packages. This may take 2–5 minutes and will print a lot of text — that is normal.

Command Prompt
C:\ResumeApp\...> pip install -r requirements.txt
✅
Verify it worked The last line should say "Successfully installed…". If you see a red error about a specific package, note the package name and contact support.
5
Load the AI model
Downloads & builds the resume-parsing model (~1 GB)
💡 Make sure you are still in the app folder in Command Prompt from Step 4. The commands below must run from that folder.
a

Download the base AI model. This is about 1 GB — keep the window open and wait until it says "success". It may take 5–10 minutes depending on your internet speed.

Command Prompt
C:\ResumeApp\...> ollama pull qwen2.5:1.5b
b

Build the custom resume-parsing model using the included Modelfile. This takes about 10 seconds.

Command Prompt
C:\ResumeApp\...> ollama create resume-parser -f Modelfile
✅
Verify it worked Run the command below. You should see resume-parser listed.
Command Prompt
C:\ResumeApp\...> ollama list
6
Configure the database
Create the app database in PostgreSQL
a

Open the .env file inside the app folder using Notepad. Find the line that starts with DATABASE_URL=. If you used a different password in Step 2, replace postgres in that line with your chosen password.

💡 If you used the default password postgres in Step 2, you do not need to change anything in this file.
b

First, create the database. In Command Prompt (still in the app folder), run:

Command Prompt
C:\ResumeApp\...> python -c "import psycopg2; conn = psycopg2.connect('postgresql://postgres:postgres@localhost:5432/postgres'); conn.autocommit = True; conn.cursor().execute('CREATE DATABASE resume_profiles'); conn.close(); print('Database created')"
c

Then create the app's tables:

Command Prompt
C:\ResumeApp\...> python -c "from app import app, db; app.app_context().push(); db.create_all(); print('Tables created')"
✅
Verify it worked Both commands should print "Database created" and "Tables created". If you see an error about the database already existing, that is fine — continue to Step 7.
7
Start the app & open it
You're almost there
a

In Command Prompt (still in the app folder), start the app:

Command Prompt — keep this window open
C:\ResumeApp\...> python app.py * Running on http://127.0.0.1:5001
b

Open your browser (Chrome or Edge recommended) and go to:

http://127.0.0.1:5001
🎉 You're done! The Resume Intelligence app should open in your browser. To close the app later, go back to the Command Prompt window and press Ctrl + C.
⚠️ Every time you want to use the app, you must open Command Prompt, navigate to the app folder, and run python app.py again.

Troubleshooting
Common problems and how to fix them
❌ "python is not recognized as a command"

Python was installed without ticking "Add Python to PATH". Uninstall Python from Windows Settings → Apps, then reinstall it — this time make sure to tick the PATH checkbox before clicking Install Now.

❌ "ollama is not recognized as a command"

Ollama may not have started. Look for the Ollama icon in your system tray (bottom-right corner). If it is not there, open the Start menu and launch Ollama. Try the command again after Ollama shows in the tray.

❌ Database connection error when starting the app

PostgreSQL may not be running. Open the Start menu, search for Services, find postgresql-x64-xx in the list, right-click it, and choose Start. Then try running the app again.

Also check that the password in the .env file matches the password you set in Step 2.

❌ App is very slow when parsing a resume

The AI model runs on your CPU, which can take 10–30 seconds per resume. This is normal. If it spins for more than 2 minutes, check that Ollama is still running in the system tray and try uploading the resume again.

❌ "No module named 'xxx'" error

A package did not install correctly. Run pip install -r requirements.txt again from the app folder. If the same error appears, note the module name and contact your IT support with the exact error message.
